# Language Support (i18n)

The site supports **6 languages** with a dropdown in the header. The chosen language is saved in the browser and used on the next visit.

---

## Supported languages

| Code | Language   |
|------|------------|
| en   | English    |
| de   | German (Deutsch) |
| hi   | Hindi (हिन्दी) |
| ta   | Tamil (தமிழ்) |
| es   | Spanish (Español) |
| ar   | Arabic (العربية) – layout switches to RTL |
| zh   | Chinese Simplified (中文) |
| fr   | French (Français) |

---

## Impact and considerations

### What works well
- **Runtime switch** – Users change language without reloading; no separate build per language.
- **Small extra size** – Only the active language JSON is loaded; others load on demand.
- **SEO** – Search engines see the default (or last used) language. For full SEO per language you’d need locale-specific URLs or server-side handling.
- **Arabic (ar)** – `dir="rtl"` is set on the document when Arabic is selected so layout flips correctly.

### Maintenance
- **New or changed text** – Add or update the key in all 6 files under `public/i18n/` (e.g. `en.json`, `hi.json`, …).
- **New language** – Add a new JSON file (e.g. `de.json`), add the language to the `languages` array in `app.component.ts`, and add the option in the template.

### Adding another language
1. Create `public/i18n/<code>.json` with the same keys as `en.json`.
2. In `app.component.ts`, add `{ code: 'xx', label: 'Language Name' }` to `languages`.
3. No template change needed; the dropdown is built from `languages`.

### RTL (Arabic)
When the user selects Arabic, the app sets `document.documentElement.setAttribute('dir', 'rtl')`. Layout and alignment follow RTL. If some sections still look wrong, add RTL-specific overrides in `styles.css` using `[dir="rtl"]` selectors.

---

## Files

- **Config:** `src/app/app.config.ts` – `provideTranslateService`, `provideTranslateHttpLoader`, `provideHttpClient`.
- **Translations:** `public/i18n/*.json` – one file per language.
- **Component:** `src/app/app.component.ts` – `TranslateService`, `setLang()`, `languages` list.
- **Template:** `src/app/app.component.html` – all user-visible strings use `{{ 'key' | translate }}`.

---

## Translating the JSON files

The English file `en.json` is the source of keys. For professional translation:
1. Send `en.json` (or export from it) to your translator.
2. They return the same structure with values in the target language.
3. Save as the correct `public/i18n/<code>.json` file.

Ensure special characters and quotes are valid JSON. For RTL languages (e.g. Arabic), no extra escaping is needed in the JSON.
